# 知识库详情页 v4.0 更新日志

**发布日期**: 2026-06-14
**基于版本**: v3.0

---

## 核心变更概览

v4.0 版本在 v3.0 的基础上进行了全面的 UI 和交互优化，重点解决了复审人操作流程、审核状态展示、审批流可视化等方面的体验问题。

### 主要亮点

1. **复审人操作直接生效** - 去除冗余弹窗，提升操作效率
2. **创建者角色定位明确** - 在"我的提交"中按初审人权限展示
3. **审核状态全面优化** - 支持审核中状态，显示真实审核人信息
4. **审批流动态展示** - 根据实际流转动态显示节点，不提前展示未到达的环节
5. **界面简化与优化** - 去除重复按钮、冗余字样，优化列显示

---

## 详细更新内容

### 1. 复审人操作直接生效（无弹窗）

**问题**：复审人作为最高审核角色，其操作应直接生效，但旧版本仍需弹出"提交审核"弹窗确认。

**改进**：
- **导入文件**：点击"导入文件"按钮后，直接提示"已导入文件并直接生效"，不弹出审核弹窗
- **更新文件**：点击"重命名"等操作后，直接执行，不弹出审核弹窗
- **删除文件**：点击"删除"后，仅弹出二次确认"确定删除该文档？"，不走审核流程

**影响范围**：
- `src/pages/workspace/knowledge-detail.tsx` - 导入按钮逻辑
- `src/pages/workspace/knowledge-detail/documents-tab.tsx` - 文档操作逻辑

**判定条件**：`isSecondReviewer && !isOwner`（纯复审人）

---

### 2. 创建者按初审人角色对待

**问题**：创建者虽具备复审能力（`canSecondReview: true`），但在"我的提交"Tab 中被当作复审人处理，导致隐藏了审核状态列和筛选器，无法查看自己提交的审核进度。

**改进**：
- 创建者在"我的提交"Tab 中按**初审人**角色展示
- 显示完整的审核状态列、审核详情列和筛选器
- 能查看自己提交的待审核、已生效、已驳回等状态

**影响范围**：
- `src/pages/workspace/knowledge-detail/my-submissions-tab.tsx`

**实现方式**：引入 `isPureSecondReviewer = isSecondReviewer && !isOwner` 判定，替代原有的 `canSecondReview`

---

### 3. "我的提交"Tab 优化

#### 3.1 差异化展示

**纯复审人（`isSecondReviewer && !isOwner`）**：
- ❌ 不显示筛选器
- ❌ 不显示"审核状态"列
- ❌ 不显示"审核详情"列
- ✅ 显示：提交时间、提交类型、提交内容、变更说明

**其他角色（创建者、初审人、维护人员）**：
- ✅ 显示筛选器（全部、待审核、审核驳回）
- ✅ 显示"审核状态"列（待初审、待复审、审核通过、审核驳回）
- ✅ 显示"审核详情"列（查看审批流按钮）
- ✅ 显示所有列

#### 3.2 显示所有记录（含已生效）

**旧版本**：过滤掉 `status === "approved"` 的记录，用户无法看到已生效的提交

**新版本**：显示所有提交记录，包括：
- 待审核（`pending_first`、`pending_second`）
- 已生效（`approved`）
- 已驳回（`rejected`）

**筛选器行为**：
- **全部**：显示所有记录
- **待审核**：只显示 `pending_first` 和 `pending_second`
- **审核驳回**：只显示 `rejected`

**影响文件**：
- `src/pages/workspace/knowledge-detail/my-submissions-tab.tsx`

### 4. 审核记录 Tab 优化

#### 4.1 列名优化

**变更**：
- "审核结果" → **"审核状态"**
- 移除"审结环节"列
- 移除重复的"提交时间"列
- "审结时间"改为"提交时间"（根据记录状态动态显示提交时间或审结时间）

#### 4.2 审核状态显示

**新增"审核中"状态**：
- 审核通过：绿色"审核通过"徽章
- 审核驳回：红色"审核驳回"徽章
- **审核中**：黄色"审核中"徽章（带时钟图标）

#### 4.3 审核人显示优化

**旧版本**：
- 审核中记录显示"待初审人审核"或"待复审人审核"（占位文字）

**新版本**：
- 审核中记录显示**当前应审核用户的真实姓名（工号）**
- `pending_first` → 从知识库配置的 `firstReviewerIds` 查询初审人
- `pending_second` → 从知识库配置的 `secondReviewerIds` 查询复审人
- 显示格式：`张强(T20180520)` 或 `李明(T20190101)、王芳(T20200215)`（多人时用顿号分隔）

**影响文件**：
- `src/pages/workspace/knowledge-detail/review-records-tab.tsx`
- 新增 `getPendingReviewerText()` 和 `formatReviewers()` 辅助函数

### 5. 审批流优化

#### 5.1 动态节点显示（不提前展示复审节点）

**旧版本问题**：
- 待初审状态时，审批流已显示"等待复审"节点
- 给用户造成困惑：明明还在初审，为什么显示复审？

**新版本**：
- 复审节点**仅在以下情况显示**：
  1. 已有复审记录（`secondReview` 存在）
  2. 当前状态是 `pending_second`（实际进入复审环节）
- **不在以下情况显示复审节点**：
  1. 还在初审环节（`pending_first`）
  2. 初审驳回（`firstReview.result === "rejected"`）
  3. 初审直接生效（`firstReview.skipSecondReview === true`）

**审批流时间线示例**：
- `pending_first`：提交 → 等待初审
- `pending_second`：提交 → 初审通过 → 等待复审
- 初审直接生效：提交 → 初审通过（直接生效）
- 初审驳回：提交 → 初审驳回

#### 5.2 初审直接生效标识

**新增**：
- 节点标题显示：**"初审通过（直接生效）"**
- 描述中附带生效版本号：`常规通知类文档，本人具备初审权限，直接生效（生效版本：v13）`

**影响文件**：
- `src/pages/workspace/knowledge-detail/approval-timeline.tsx`（新增可复用组件）
- `src/pages/workspace/knowledge-detail/approval-flow-dialog.tsx`（重构为使用共享组件）

#### 5.3 详情弹窗底部显示审批流

**新增**：
- 在审核详情弹窗底部新增"审批流"时间线
- 只读模式下展示完整的提交 → 初审 → 复审流程
- 与"查看审批流"弹窗使用同一套时间线渲染逻辑，保证一致性

**影响文件**：
- `src/pages/workspace/knowledge-detail/review-detail-dialog.tsx`

## 升级指南

### 从 v3.0 升级到 v4.0

#### 1. 代码变更
所有代码变更已实现，无需额外操作。构建命令：
```bash
npm run build
```

#### 2. 数据兼容性
v4.0 完全兼容 v3.0 的数据结构，无需数据迁移。

#### 3. 行为变更说明

**复审人用户需要注意**：
- 导入、更新、删除操作不再弹出"提交审核"弹窗
- 操作直接生效，无需等待审核

**创建者用户需要注意**：
- "我的提交"Tab 现在显示完整的审核流程信息
- 可以查看自己提交的所有记录（包括已生效的）

**所有用户**：
- "我的提交"Tab 现在包含已生效的记录
- 审核记录 Tab 现在包含审核中的记录
- 审批流时间线根据实际流转动态显示，不会提前展示未到达的节点
