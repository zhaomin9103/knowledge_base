关键规则说明

  • 维护人员的任何变更（新增 / 更新 / 删除）都必须走审核，提交后状态为 pending，进入创建者 / 管理员的「待审核」队列。
  • 创建者 / 管理员对文件的操作直接生效，不进入审核流（权限判定为 canReview 而非 canSubmit）。
  • 审核通过：系统分配 appliedVersion，文件落入「文件列表」，并派生一条版本记录进入「版本记录」。审核通过的文件不再在维护人员「我的提交」中重复显示。
  • 审核驳回：必须填写驳回原因，维护人员可在「我的提交 · 审核驳回」中查看原因，修改后重新提交。
  • 可追溯性：所有通过 / 驳回写入「审核记录」，所有生效变更写入「操作记录」。
