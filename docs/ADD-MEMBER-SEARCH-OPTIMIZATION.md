# 添加成员弹窗优化 - 搜索式交互

**更新日期**: 2026-06-14  
**版本**: v4.1

---

## 📋 优化概述

重新设计添加成员弹窗的交互流程，从"浏览选择"改为"搜索选择"模式，提升大量用户场景下的操作效率。

---

## ✨ 核心功能

### 1. 搜索式交互

**旧版本**：
- 显示所有可添加用户（不含已是成员的）
- 用户需要滚动浏览列表找人
- 支持按组织筛选

**新版本**：
- 列表默认为空，显示搜索提示
- 用户必须输入搜索关键词才会显示结果
- 仅支持姓名/工号搜索（精确查找）

### 2. 搜索功能

**支持的搜索方式**：
- ✅ **姓名搜索**：输入"张三"显示所有名字包含"张三"的用户
- ✅ **工号搜索**：输入"T20200301"或"001"显示工号匹配的用户
- ✅ **部分匹配**：输入"张"显示所有姓张的用户
- ❌ **不支持组织搜索**：输入"学生工作处"不会有结果

**搜索逻辑**：
```typescript
const keyword = searchKeyword.trim().toLowerCase()

// 按姓名或工号匹配
const nameMatch = user.name.toLowerCase().includes(keyword)
const idNoMatch = user.idNo.toLowerCase().includes(keyword)

return nameMatch || idNoMatch
```

### 3. 已选人数显示

标题栏右侧实时显示"已选 X 人"：

```
┌─────────────────────────────────────────┐
│ [图标] 添加成员          已选 3 人      │
└─────────────────────────────────────────┘
```

- 未选择时：不显示
- 选择后：显示品牌色文字"已选 X 人"

### 4. UI 状态

#### 空状态（未输入搜索）
```
┌──────────────────────────┐
│      [搜索图标]          │
│   请输入姓名或工号搜索成员  │
└──────────────────────────┘
```

#### 无结果状态
```
┌──────────────────────────┐
│    未找到匹配的成员       │
│    请尝试其他关键词       │
└──────────────────────────┘
```

#### 搜索结果状态
```
☑ [头像] 张三  T20200301
         学生工作处         [教师]

☐ [头像] 张强  T20180520
         学生工作处         [教师]
```

## 🎨 交互流程

### 完整操作流程

1. **打开弹窗**
   - 点击"添加成员"按钮
   - 弹窗显示，列表为空状态

2. **搜索成员**
   - 在搜索框输入姓名或工号（如"张三"或"T20200301"）
   - 实时显示匹配的搜索结果
   - 点击搜索框右侧的 ✕ 可清空

3. **选择成员**
   - 点击用户卡片勾选/取消勾选
   - 标题栏显示"已选 X 人"
   - 支持多选

4. **分配角色**
   - 在"分配角色"下拉框选择：维护人员/初审人/复审人
   - 所有选中的成员将被分配相同的角色

5. **确认添加**
   - 点击"确认添加 (X)"按钮
   - 添加成功后弹窗关闭

## 💡 设计优势

### 1. 提升效率
- **旧版本**：用户池有上百个用户时，需要滚动浏览
- **新版本**：直接输入工号或姓名，立即找到目标用户

### 2. 减少误操作
- **旧版本**：长列表容易误点
- **新版本**：明确搜索目标，减少误选

### 3. 清晰反馈
- 实时显示"已选 X 人"
- 三种状态清晰区分（空状态/无结果/有结果）
- 选中用户有背景高亮

### 4. 更好的扩展性
- 用户数量增加时，体验不会下降
- 可轻松扩展到支持更多搜索条件（如按角色筛选）

## 🔧 技术实现

### 搜索过滤逻辑

```typescript
const searchResults = useMemo(() => {
  const keyword = searchKeyword.trim().toLowerCase()

  // 如果没有输入搜索关键词，返回空数组
  if (!keyword) {
    return []
  }

  return MOCK_USERS.filter((user) => {
    // 排除已是成员的用户
    if (existingMemberIds.includes(user.id)) {
      return false
    }

    // 按姓名或工号搜索
    const nameMatch = user.name.toLowerCase().includes(keyword)
    const idNoMatch = user.idNo.toLowerCase().includes(keyword)

    return nameMatch || idNoMatch
  })
}, [searchKeyword, existingMemberIds])
```
