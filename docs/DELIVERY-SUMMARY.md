## 🎯 解决的核心问题

### 问题 1：复审人操作冗余弹窗
**现象**: 复审人作为最高审核角色，导入/更新/删除文件时仍需弹出"提交审核"确认弹窗  
**影响**: 降低操作效率，造成困惑  
**解决**: 
- 导入按钮：纯复审人点击后直接生效，提示"已导入文件并直接生效"
- 文档操作：更新/删除直接执行，不弹出审核弹窗

### 问题 2：创建者角色定位混淆
**现象**: 创建者在"我的提交"Tab 中被当作复审人，隐藏了审核状态和筛选器  
**影响**: 创建者无法查看自己提交的审核进度  
**解决**: 引入 `isPureSecondReviewer` 精确判定，创建者按初审人角色展示完整信息

### 问题 3："我的提交"无法查看已生效记录
**现象**: 旧版本过滤掉 `approved` 状态，用户看不到已生效的提交  
**影响**: 无法追溯历史提交  
**解决**: 显示所有提交记录（含已生效），筛选器提供"全部/待审核/审核驳回"选项

### 问题 4：审核记录中审核中状态不明确
**现象**: 
- 审核记录 Tab 只显示已审结记录
- 审核中记录显示"待初审人审核"占位文字，不显示真实审核人
**影响**: 无法快速了解当前待审核记录和负责人  
**解决**:
- 新增"审核中"状态（黄色徽章）
- 从知识库配置查询真实审核人并显示姓名（工号）

### 问题 5：审批流提前显示未到达的节点
**现象**: 待初审状态时，审批流已显示"等待复审"节点  
**影响**: 造成用户困惑，误以为已进入复审环节  
**解决**: 复审节点仅在实际进入复审环节或已有复审记录时才显示

### 问题 6：审核详情弹窗缺少审批流信息
**现象**: 查看审核详情时，需要关闭弹窗再点击"查看审批流"才能看到流程  
**影响**: 操作繁琐  
**解决**: 详情弹窗底部直接显示审批流时间线

## 🔧 技术实现亮点

### 1. 组件化架构
抽取可复用的 `ApprovalTimeline` 组件，统一管理审批流时间线渲染逻辑：
- `approval-flow-dialog.tsx` - 审批流弹窗
- `review-detail-dialog.tsx` - 审核详情弹窗底部
- 未来可扩展到其他需要展示审批流的场景

### 2. 精确角色判定
引入 `isPureSecondReviewer = isSecondReviewer && !isOwner` 判定逻辑，解决创建者角色定位问题：
- 纯复审人：直接生效，简化界面
- 创建者：虽具备复审能力，但在提交场景按初审人对待

### 3. 动态节点显示算法
审批流时间线根据以下条件动态决定是否显示复审节点：
```typescript
const firstRejected = review.firstReview?.result === "rejected"
const shouldShowSecondReview = 
  !firstRejected && 
  (review.secondReview || review.status === "pending_second")
```

### 4. 审核人信息查询
从知识库配置实时查询当前应审核人：
```typescript
function getPendingReviewerText(record: ReviewRequest): string {
  const kb = KNOWLEDGE_BASES.find((k) => k.id === record.kbId)
  if (record.status === "pending_first") {
    return formatReviewers(kb.firstReviewerIds)
  }
  if (record.status === "pending_second") {
    return formatReviewers(kb.secondReviewerIds)
  }
  return "—"
}
```

## 📝 用户文档

### 角色操作指南

#### 复审人
1. **导入文件**: 点击"导入文件"按钮，文件直接生效
2. **更新文件**: 点击"重命名"等操作，直接执行
3. **删除文件**: 点击"删除"，确认后直接删除
4. **我的提交**: 查看所有提交记录（简化视图）

#### 创建者
1. **导入文件**: 点击"导入文件"，选择"确定生效"或"提交复审"
2. **更新/删除**: 填写变更说明，选择生效方式
3. **我的提交**: 查看完整审核流程（含筛选器、审核状态）
4. **成员管理**: 添加/移除成员，变更角色

#### 初审人
1. **导入文件**: 点击"导入文件"，选择"确定生效"或"提交复审"
2. **审核提交**: 在"待初审"Tab 中审核，可选择直接生效或提交复审
3. **我的提交**: 查看完整审核流程
4. **审核记录**: 查看所有提交的审核记录

#### 维护人员
1. **导入文件**: 点击"导入文件"，填写变更说明，提交审核
2. **更新/删除**: 填写变更说明，提交审核
3. **我的提交**: 查看自己提交的审核进度
4. **审核记录**: 查看所有提交的审核记录
